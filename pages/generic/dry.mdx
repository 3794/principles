# DRY

## Don't Repeat Yourself

반복되는 코드를 작성하지 마라.

Origin

Andy Hunt와 Dave Thomas가 저술한 "The Pragmatic Programmer"에서 처음 소개된 원칙이다.

Why

- 코드 중복은 유지보수를 어렵게 만든다
- 버그 발생 가능성을 높인다
- 코드의 일관성을 해친다

Resources

- [Don't Repeat Yourself (wikipedia.org)](https://en.wikipedia.org/wiki/Don%27t_repeat_yourself)
- [DRY 원칙 (johngrib.github.io)](https://johngrib.github.io/wiki/jargon/dry-principle/)

Example

```typescript
// 나쁜 예시: 반복되는 코드
const UserList = () => {
  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);

  const fetchUsers = async () => {
    setLoading(true);
    try {
      const response = await fetch("/api/users");
      const data = await response.json();
      setUsers(data);
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  };

  useEffect(() => {
    fetchUsers();
  }, []);

  if (loading) return <div>로딩 중...</div>;
  if (error) return <div>에러: {error}</div>;

  return (
    <div>
      {users.map((user) => (
        <div key={user.id}>
          <h3>{user.name}</h3>
          <p>{user.email}</p>
        </div>
      ))}
    </div>
  );
};

const PostList = () => {
  const [posts, setPosts] = useState([]);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);

  const fetchPosts = async () => {
    setLoading(true);
    try {
      const response = await fetch("/api/posts");
      const data = await response.json();
      setPosts(data);
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  };

  useEffect(() => {
    fetchPosts();
  }, []);

  if (loading) return <div>로딩 중...</div>;
  if (error) return <div>에러: {error}</div>;

  return (
    <div>
      {posts.map((post) => (
        <div key={post.id}>
          <h3>{post.title}</h3>
          <p>{post.content}</p>
        </div>
      ))}
    </div>
  );
};

// 좋은 예시: 재사용 가능한 컴포넌트와 훅으로 중복 제거
// 데이터 페칭을 위한 커스텀 훅
const useData = <T>(url: string) => {
  const [data, setData] = useState<T[]>([]);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<Error | null>(null);

  useEffect(() => {
    const fetchData = async () => {
      setLoading(true);
      try {
        const response = await fetch(url);
        const result = await response.json();
        setData(result);
      } catch (err) {
        setError(err as Error);
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, [url]);

  return { data, loading, error };
};

// 재사용 가능한 로딩 컴포넌트
const LoadingSpinner = () => <div>로딩 중...</div>;

// 재사용 가능한 에러 컴포넌트
const ErrorMessage = ({ message }: { message: string }) => (
  <div>에러: {message}</div>
);

// 재사용 가능한 리스트 컴포넌트
const DataList = <T extends { id: number }>({
  data,
  renderItem,
}: {
  data: T[];
  renderItem: (item: T) => React.ReactNode;
}) => (
  <div>
    {data.map((item) => (
      <div key={item.id}>{renderItem(item)}</div>
    ))}
  </div>
);

// 사용자 리스트 컴포넌트
const UserList = () => {
  const { data: users, loading, error } = useData<User>("/api/users");

  if (loading) return <LoadingSpinner />;
  if (error) return <ErrorMessage message={error.message} />;

  return (
    <DataList
      data={users}
      renderItem={(user) => (
        <>
          <h3>{user.name}</h3>
          <p>{user.email}</p>
        </>
      )}
    />
  );
};

// 게시물 리스트 컴포넌트
const PostList = () => {
  const { data: posts, loading, error } = useData<Post>("/api/posts");

  if (loading) return <LoadingSpinner />;
  if (error) return <ErrorMessage message={error.message} />;

  return (
    <DataList
      data={posts}
      renderItem={(post) => (
        <>
          <h3>{post.title}</h3>
          <p>{post.content}</p>
        </>
      )}
    />
  );
};
```

이 예제에서 DRY 원칙을 적용한 주요 개선사항:

1. **커스텀 훅 추출**

   - `useData` 훅으로 데이터 페칭 로직을 재사용

2. **공통 컴포넌트 추출**

   - `LoadingSpinner`: 로딩 상태 표시
   - `ErrorMessage`: 에러 상태 표시
   - `DataList`: 데이터 리스트 렌더링

이렇게 DRY 원칙을 적용하면:

- 코드 중복이 제거됨
- 유지보수가 쉬워짐
- 버그 발생 가능성이 감소
- 코드의 재사용성이 높아짐
