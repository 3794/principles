# Modular Reasoning(모듈화된 추론)

**모듈화된 추론(Modular Reasoning)** 은 프로그램을 각 모듈 단위로 분석하고 검증할 수 있도록 하는 방법론이다.

---

### 정의

모듈화된 추론은 프로그램의 특정 부분이나 모듈을 분석할 때, 그 부분이 다른 모듈과 어떻게 상호작용하는지에 대한 세부 정보를 알 필요 없이,
해당 모듈의 명세(specification)만을 기반으로 추론할 수 있도록 한다. 즉, 모듈의 내부 구현을 참조하지 않고도 외부 명세를 통해 correctness를 검증하는 방식이다.

### Local Reasoning과의 차이점

Local Reasoning은 코드 내부에서 사용하는 자원/상태에 대한 reasoning, Modular Reasoning은 모듈 간 인터페이스를 통한 reasoning에 초점을 맞춘다.

#### Example

```typescript
function increment(x: number) {
  return x + 1;
}

function useIncrement(y: number) {
  return increment(y) * 2;
}
```

##### Local Reasoning 관점

```typescript
function increment(x: number) {
  return x + 1;
}
```

이 함수는 x만 다루기 때문에 `x + 1`만 reasoning 하면 된다.
즉, x에만 집중해서 이 함수의 동작을 증명할 수 있다면 local reasoning이다.

##### Modular Reasoning 관점

```typescript
function useIncrement(y: number) {
  return increment(y) * 2;
}
```

여기서 increment의 구현을 몰라도 "increment는 x를 1증가시킨다"는 스펙만 알면 reasoning 할 수 있다.
그러면 `useIncrement(3)`은 `4 * 2 = 8`이라는걸 추론할 수 있다.
즉, 모듈간 인터페이스(스펙)을 기반으로 reasoning 하는것이 modular reasoning이다.
