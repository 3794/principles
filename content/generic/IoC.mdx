# IoC, Inversion of Control

Inversion of Control(제어역전)이란 프로그램이 외부 소스(예: 프레임워크)로부터 제어흐름을 받는 디자인 원칙을 말한다. 줄여서 IoC라고 부른다.
GUI 환경과 웹서버 애플리케이션에서 계속 사용되어 왔다. 웹 서버 애플리케이션 프레임워크에서는 디스패치를 일반적으로 라우팅이라고 하며,
핸들러는 엔드포인트라고 부른다.

[할리우드 원칙](/generic/hollywood-principle)으로도 불린다.

웹브라우저는 Inversion of Control 구현을 통해 DOM 이벤트를 처리한다. 개발자는 `addEventListener`를 사용하여 callback을 등록한다.

```html
<!DOCTYPE html>
<html>
  <body>
    <span id="output"></span>

    <script>
      var registeredListener = function () {
        document.getElementById("output").innerHTML =
          "<large>The registered listener was called.</large>";
      };
      document.addEventListener("click", registeredListener, true);

      document.getElementById("output").innerHTML = `<large>
            The event handler has been registered. 
            If you click the page, your web browser will call the event handler.
          </large>`;
    </script>
  </body>
</html>
```

### 스프링프레임워크에서 IoC의 의미

Java커뮤니티의 Spring Framework에서 사용되는 IoC는 IoC 컨테이너에서 발생하는 종속성 주입 패턴을 지칭하기 위해 사용되는 용어로,
프레임워크에 애플리케이션 코드의 제어 흐름을 부여하는 원래 의미와는 다르다.
