# Code For The Maintainer

코드나 시스템을 지속적으로 관리하고 유지하는 사람([maintainer](/concepts/Maintainer))을 위해서 코드를 작성하는 것은 좋은 생각이다. 왜냐면 유지보수가 가장 비용이 많이 드는 단계이기 때문이다.

시스템에 대한 지식이 적은 사람이 이후에 코드를 변경할 것이라는 점을 염두에 둔다면, 더 명확한 코드를 작성할 뿐만 아니라 실수를 하더라도 피해를 최소화하기 위해 제한적인 코드를 작성하게 된다.
