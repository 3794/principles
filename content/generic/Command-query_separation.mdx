# CQS, Command Query Separation

[Eiffel](/concepts/Eiffel) 프로그래밍 언어 개발 과정에서 [Bertrand Meyer](/people/Bertrand_Meyer)가 제안함.

이 원칙은 모든 메서드는 **action을 수행하는 command** 이거나, **caller에게 데이터를 반환하는 query** 중 하나여야 하며, 두 가지를 동시에 수행해서는 안 된다고 명시한다.
다시 말해, 질문을 하는 행위가 답변을 변경해서는 안 된다는 것이다.

좀 더 엄밀하게 말하면, 메서드는 [referential transparency](/concepts/Referential_transparency)을 가지며 [side effect](/concepts/Side_effect)가 없을 때에만 값을 반환해야 한다.
