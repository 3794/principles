# Code bloat (코드 부풀림)

**Code bloat** 은 소프트웨어 개발에서 코드 크기가 불필요하게 증가하여 성능 저하와 유지보수 어려움을 초래하는 현상을 말한다.

---

### 코드 부풀림의 문제점

1. 가독성이 저하 되어 코드의 목적과 실제 작동을 이해하기 어려워 진다. 이는 버그 증가와 개발자의 작업 효율 저하로 이어짐.
2. 프로그렘이 커지면서 배포와 실행 시간이 증가하고 효율성이 떨어지며 성능이 저하될 수 있다.

### Example

이 코드는 불필요한 변수 선언과 비효율적인 문자열 연결을 포함하고 있다.

```typescript
type MapImage = (
  size: number,
  zoom: number,
  marker: { latitude: number; longitude: number }
) => string;

const mapImage: MapImage = (size, zoom, marker) => {
  const URL = "https://map.example.com/api";
  const strSize = "&size=" + size;
  const strZoom = "&zoom=" + zoom;
  let strFinalImage = "";

  let strURL = URL + "?center=";
  strURL += marker.latitude;
  strURL += ",";
  strURL += marker.longitude;
  strURL += strSize;
  strURL += strZoom;

  const strHTMLStart = '<img src="';
  const strHTMLEnd = '" alt="The map"/>';

  strFinalImage = strHTMLStart + strURL + strHTMLEnd;
  return strFinalImage;
};
```

불필요한 변수와 중복된 로직을 제거하여 간결하게 작성된 코드

```typescript
const mapImage: MapImage = (size, zoom, marker) => {
  const URL = "https://map.example.com/api";
  const fullURL = `${URL}?center=${marker.latitude},${marker.longitude}&size=${size}&zoom=${zoom}`;

  return `<img src="${fullURL}" alt="The map" />`;
};
```
