# Programing paradigm

프로그램을 작성하는 방식이나 사고방식(스타일)을 말한다.
쉽게 말하면 "코드를 어떻게 조직하고 문제를 어떻게 풀 것인가에 대한 철학"이다.

- Imperative Programming(명령형 프로그래밍)
  - [Object-Oriented Programming(객체 지향 프로그래밍)](/concepts/Object-oriented_programming)
    - Class based(클래스 기반)
    - Prototype based(프로토타입 기반)
  - Procedural Programming(절차형 프로그래밍)
- Declarative Programming(선언형 프로그래밍)
  - [Functional Programming(함수형 프로그래밍)](/concepts/Functional_programming)
  - [Reactive Programming(반응형 프로그래밍)](/concepts/Reactive_programming)
  - [Functional Reactive Programming(함수형 반응형 프로그래밍)](/concepts/Functional_Reactive_programming)
- Concurrent Programming(동시성 프로그래밍)
