# Dependency (의존성)

> 제가 Dependency’라는 용어를 사용할 때는, [UML](/concepts/UML)에서 정의된 의미로 사용합니다.
> 예를 들어, UI 모듈의 코드가 도메인 모듈의 코드를 참조한다면-함수를 호출하거나, 데이터를 사용하거나, 도메인 모듈에 정의된 타입을 사용할 때-UI 모듈은 도메인 모듈에 의존한다고 할 수 있습니다.
> 만약 누군가 도메인 모듈을 변경한다면, UI 모듈도 변경이 필요할 가능성이 있습니다.
> 의존성은 단방향입니다. 즉, 일반적으로 UI 모듈이 도메인 모듈에 의존하지, 그 반대는 아닙니다.
> 만약 도메인 모듈도 UI 모듈에 의존하게 된다면, 그때는 두 번째 의존성이 생기는 것입니다. - [Martin Fowler](/people/Martin_Fowler) [^1]
