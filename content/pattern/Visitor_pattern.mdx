# Visitor pattern

Visitor(방문자) 패턴은 [OOP](/concepts/Object-oriented_programming)에서 "데이터 구조(객체)"와 "알고리즘"을 분리해, 객체 구조를 변경하지 않고 새로운 알고리즘을 추가할 수 있도록 설계된 패턴이다.

## Core Concept(핵심 개념)

- **Element** - 방문받을 객체들 (방문자가 수행할 작업의 대상)
- **Visitor** - 각 Element에 수행할 작업을 정의하는 인터페이스
- **구조와 알고리즘 분리** - 새로운 알고리즘을 추가할 때 Element 클래스 수정 불필요

## Simple Example(간단한 예제)

도형 시스템에서 넓이 계산, 둘레 계산 등 다양한 작업을 추가하는 예제이다.

### 1. Element 정의 (도형들)

```java
// 방문자가 방문받을 도형 인터페이스
interface Shape {
  void accept(ShapeVisitor visitor);
}

// 구체적인 도형들
class Circle implements Shape {
  private double radius;

  public Circle(double radius) {
    this.radius = radius;
  }

  public double getRadius() {
    return radius;
  }

  @Override
  public void accept(ShapeVisitor visitor) {
    visitor.visitCircle(this);
  }
}

class Rectangle implements Shape {
  private double width;
  private double height;

  public Rectangle(double width, double height) {
    this.width = width;
    this.height = height;
  }

  public double getWidth() {
    return width;
  }

  public double getHeight() {
    return height;
  }

  @Override
  public void accept(ShapeVisitor visitor) {
    visitor.visitRectangle(this);
  }
}

class Triangle implements Shape {
  private double sideA;
  private double sideB;
  private double sideC;

  public Triangle(double sideA, double sideB, double sideC) {
    this.sideA = sideA;
    this.sideB = sideB;
    this.sideC = sideC;
  }

  public double getSideA() { return sideA; }
  public double getSideB() { return sideB; }
  public double getSideC() { return sideC; }

  @Override
  public void accept(ShapeVisitor visitor) {
    visitor.visitTriangle(this);
  }
}
```

### 2. Visitor 인터페이스 정의

```java
// 모든 방문자가 구현해야 할 인터페이스
interface ShapeVisitor {
  void visitCircle(Circle circle);
  void visitRectangle(Rectangle rectangle);
  void visitTriangle(Triangle triangle);
}
```

### 3. 구체적인 Visitor 구현 (넓이 계산)

```java
class AreaCalculator implements ShapeVisitor {
  private double totalArea = 0;

  public double getTotalArea() {
    return totalArea;
  }

  @Override
  public void visitCircle(Circle circle) {
    totalArea += Math.PI * circle.getRadius() * circle.getRadius();
  }

  @Override
  public void visitRectangle(Rectangle rectangle) {
    totalArea += rectangle.getWidth() * rectangle.getHeight();
  }

  @Override
  public void visitTriangle(Triangle triangle) {
    // 헤론의 공식 사용
    double s = (triangle.getSideA() + triangle.getSideB() + triangle.getSideC()) / 2;
    totalArea += Math.sqrt(
      s *
        (s - triangle.getSideA()) *
        (s - triangle.getSideB()) *
        (s - triangle.getSideC())
    );
  }
}
```

### 4. 다른 Visitor 구현 (둘레 계산)

```java
class PerimeterCalculator implements ShapeVisitor {
  private double totalPerimeter = 0;

  public double getTotalPerimeter() {
    return totalPerimeter;
  }

  @Override
  public void visitCircle(Circle circle) {
    totalPerimeter += 2 * Math.PI * circle.getRadius();
  }

  @Override
  public void visitRectangle(Rectangle rectangle) {
    totalPerimeter += 2 * (rectangle.getWidth() + rectangle.getHeight());
  }

  @Override
  public void visitTriangle(Triangle triangle) {
    totalPerimeter += triangle.getSideA() + triangle.getSideB() + triangle.getSideC();
  }
}
```

### 5. 사용 예제

```java
public class VisitorPatternDemo {
  public static void main(String[] args) {
    // 도형 컬렉션
    List<Shape> shapes = Arrays.asList(
      new Circle(5),
      new Rectangle(4, 6),
      new Triangle(3, 4, 5)
    );

    // 넓이 계산
    AreaCalculator areaVisitor = new AreaCalculator();
    for (Shape shape : shapes) {
      shape.accept(areaVisitor);
    }
    System.out.println("Total Area: " + areaVisitor.getTotalArea());

    // 둘레 계산 (새로운 알고리즘, 도형 클래스 수정 없음)
    PerimeterCalculator perimeterVisitor = new PerimeterCalculator();
    for (Shape shape : shapes) {
      shape.accept(perimeterVisitor);
    }
    System.out.println("Total Perimeter: " + perimeterVisitor.getTotalPerimeter());
  }
}
```

## Key Points(핵심 포인트)

- **구조와 알고리즘 분리** - 도형(Element)을 수정하지 않고 새로운 계산(Visitor) 추가 가능
- **확장성** - 새로운 Visitor를 만드는 것은 간단하지만, 새로운 도형 추가는 기존 Visitor 수정 필요
- **이중 디스패치** - `accept()` 메서드를 통해 런타임에 올바른 Visitor 메서드 호출
- **복잡한 작업에 유용** - 여러 도형에 복잡한 작업을 수행할 때 코드 가독성 향상

## 언제 사용할 것인가

- 객체 구조가 자주 변하지 않지만, 수행할 작업이 자주 추가되는 경우
- 여러 객체 타입에 대해 다양한 작업을 수행해야 하는 경우
- 관련 작업들을 한곳에 모아서 관리하고 싶은 경우
