# Visitor pattern

Visitor(방문자) 패턴은 [OOP](/concepts/Object-oriented_programming)에서 "데이터 구조(객체)"와 "알고리즘"을 분리해, 객체 구조를 변경하지 않고 새로운 알고리즘을 추가할 수 있도록 설계된 패턴이다.

## Core Concept(핵심 개념)

- **Element** - 방문받을 객체들 (방문자가 수행할 작업의 대상)
- **Visitor** - 각 Element에 수행할 작업을 정의하는 인터페이스
- **구조와 알고리즘 분리** - 새로운 알고리즘을 추가할 때 Element 클래스 수정 불필요

## Simple Example(간단한 예제)

도형 시스템에서 넓이 계산, 둘레 계산 등 다양한 작업을 추가하는 예제이다.

### 1. Element 정의 (도형들)

```typescript
// 방문자가 방문받을 도형 인터페이스
interface Shape {
  accept(visitor: ShapeVisitor): void;
}

// 구체적인 도형들
class Circle implements Shape {
  constructor(public radius: number) {}

  accept(visitor: ShapeVisitor): void {
    visitor.visitCircle(this);
  }
}

class Rectangle implements Shape {
  constructor(public width: number, public height: number) {}

  accept(visitor: ShapeVisitor): void {
    visitor.visitRectangle(this);
  }
}

class Triangle implements Shape {
  constructor(
    public sideA: number,
    public sideB: number,
    public sideC: number
  ) {}

  accept(visitor: ShapeVisitor): void {
    visitor.visitTriangle(this);
  }
}
```

### 2. Visitor 인터페이스 정의

```typescript
// 모든 방문자가 구현해야 할 인터페이스
interface ShapeVisitor {
  visitCircle(circle: Circle): void;
  visitRectangle(rectangle: Rectangle): void;
  visitTriangle(triangle: Triangle): void;
}
```

### 3. 구체적인 Visitor 구현 (넓이 계산)

```typescript
class AreaCalculator implements ShapeVisitor {
  private totalArea: number = 0;

  getTotalArea(): number {
    return this.totalArea;
  }

  visitCircle(circle: Circle): void {
    this.totalArea += Math.PI * circle.radius * circle.radius;
  }

  visitRectangle(rectangle: Rectangle): void {
    this.totalArea += rectangle.width * rectangle.height;
  }

  visitTriangle(triangle: Triangle): void {
    // 헤론의 공식 사용
    const s =
      (triangle.sideA + triangle.sideB + triangle.sideC) / 2;
    this.totalArea += Math.sqrt(
      s *
        (s - triangle.sideA) *
        (s - triangle.sideB) *
        (s - triangle.sideC)
    );
  }
}
```

### 4. 다른 Visitor 구현 (둘레 계산)

```typescript
class PerimeterCalculator implements ShapeVisitor {
  private totalPerimeter: number = 0;

  getTotalPerimeter(): number {
    return this.totalPerimeter;
  }

  visitCircle(circle: Circle): void {
    this.totalPerimeter += 2 * Math.PI * circle.radius;
  }

  visitRectangle(rectangle: Rectangle): void {
    this.totalPerimeter += 2 * (rectangle.width + rectangle.height);
  }

  visitTriangle(triangle: Triangle): void {
    this.totalPerimeter +=
      triangle.sideA + triangle.sideB + triangle.sideC;
  }
}
```

### 5. 사용 예제

```typescript
// 도형 컬렉션
const shapes: Shape[] = [
  new Circle(5),
  new Rectangle(4, 6),
  new Triangle(3, 4, 5),
];

// 넓이 계산
const areaVisitor = new AreaCalculator();
shapes.forEach((shape) => shape.accept(areaVisitor));
console.log(`Total Area: ${areaVisitor.getTotalArea()}`);

// 둘레 계산 (새로운 알고리즘, 도형 클래스 수정 없음)
const perimeterVisitor = new PerimeterCalculator();
shapes.forEach((shape) => shape.accept(perimeterVisitor));
console.log(`Total Perimeter: ${perimeterVisitor.getTotalPerimeter()}`);
```

## Key Points(핵심 포인트)

- **구조와 알고리즘 분리** - 도형(Element)을 수정하지 않고 새로운 계산(Visitor) 추가 가능
- **확장성** - 새로운 Visitor를 만드는 것은 간단하지만, 새로운 도형 추가는 기존 Visitor 수정 필요
- **이중 디스패치** - `accept()` 메서드를 통해 런타임에 올바른 Visitor 메서드 호출
- **복잡한 작업에 유용** - 여러 도형에 복잡한 작업을 수행할 때 코드 가독성 향상

## 언제 사용할 것인가

- 객체 구조가 자주 변하지 않지만, 수행할 작업이 자주 추가되는 경우
- 여러 객체 타입에 대해 다양한 작업을 수행해야 하는 경우
- 관련 작업들을 한곳에 모아서 관리하고 싶은 경우
