# Event Sourcing

Event Sourcing은 애플리케이션의 상태 변화를 직접 저장하는 대신, 모든 변화를 유발하는 **이벤트들을 시간순으로 저장**하는 아키텍처 패턴이다. 이벤트 스트림으로부터 현재 상태를 언제든지 재구성할 수 있다.

## Core Concepts(핵심 개념)

- **Event Store** - 모든 이벤트를 불변(immutable)의 로그로 저장하는 중앙 저장소
- **Event Stream** - 시간순으로 정렬된 이벤트들의 시퀀스
- **State Reconstruction** - 이벤트들을 재생해서 어떤 시점의 상태든 복원 가능
- **Single Source of Truth** - 이벤트 스트림이 유일한 진실

## Key Benefits(주요 이점)

- 완전한 감사 추적(audit trail) 제공 - 모든 변화의 기록이 남음
- 시간 여행 디버깅 - 과거의 어떤 시점 상태도 확인 가능
- 동시성 처리 용이 - 이벤트 기반으로 동시 작업 처리
- 이벤트 재생 - 버그 수정 후 과거 이벤트를 다시 처리
- CQRS와 결합하여 강력한 시스템 구축

## Simple Example(간단한 예제)

은행 계좌 시스템을 Event Sourcing으로 구현한 예제이다.

### 1. Events 정의

```typescript
// src/events/DomainEvent.ts
interface DomainEvent {
  eventId: string;
  eventType: string;
  aggregateId: string;
  timestamp: Date;
  data: any;
}

// src/events/AccountEvents.ts
export class AccountCreatedEvent implements DomainEvent {
  eventId = Math.random().toString();
  eventType = "AccountCreated";
  timestamp = new Date();

  constructor(
    public aggregateId: string,
    public data: { accountHolder: string; initialBalance: number }
  ) {}
}

export class MoneyDepositedEvent implements DomainEvent {
  eventId = Math.random().toString();
  eventType = "MoneyDeposited";
  timestamp = new Date();

  constructor(
    public aggregateId: string,
    public data: { amount: number }
  ) {}
}

export class MoneyWithdrawnEvent implements DomainEvent {
  eventId = Math.random().toString();
  eventType = "MoneyWithdrawn";
  timestamp = new Date();

  constructor(
    public aggregateId: string,
    public data: { amount: number }
  ) {}
}
```

### 2. Event Store

```typescript
// src/eventstore/EventStore.ts
export class EventStore {
  private events: DomainEvent[] = [];

  append(event: DomainEvent): void {
    this.events.push(event);
  }

  getEventsByAggregateId(aggregateId: string): DomainEvent[] {
    return this.events.filter((e) => e.aggregateId === aggregateId);
  }

  getAllEvents(): DomainEvent[] {
    return [...this.events];
  }
}
```

### 3. Aggregate (엔티티)

```typescript
// src/aggregates/Account.ts
export class Account {
  id: string;
  accountHolder: string;
  balance: number = 0;
  version: number = 0; // 이벤트 버전 추적

  private changes: DomainEvent[] = [];

  constructor(id: string) {
    this.id = id;
  }

  // 이벤트 로드
  loadFromHistory(events: DomainEvent[]): void {
    events.forEach((event) => this.applyEvent(event));
  }

  // 비즈니스 로직
  createAccount(accountHolder: string, initialBalance: number): void {
    const event = new AccountCreatedEvent(this.id, {
      accountHolder,
      initialBalance,
    });
    this.applyEvent(event);
    this.changes.push(event);
  }

  deposit(amount: number): void {
    if (amount <= 0) throw new Error("Amount must be positive");

    const event = new MoneyDepositedEvent(this.id, { amount });
    this.applyEvent(event);
    this.changes.push(event);
  }

  withdraw(amount: number): void {
    if (amount > this.balance) throw new Error("Insufficient balance");

    const event = new MoneyWithdrawnEvent(this.id, { amount });
    this.applyEvent(event);
    this.changes.push(event);
  }

  // 이벤트 적용 (상태 업데이트)
  private applyEvent(event: DomainEvent): void {
    switch (event.eventType) {
      case "AccountCreated":
        this.accountHolder = event.data.accountHolder;
        this.balance = event.data.initialBalance;
        break;
      case "MoneyDeposited":
        this.balance += event.data.amount;
        break;
      case "MoneyWithdrawn":
        this.balance -= event.data.amount;
        break;
    }
    this.version++;
  }

  // 변경사항 가져오기
  getUncommittedChanges(): DomainEvent[] {
    return this.changes;
  }

  // 변경사항 커밋
  markChangesAsCommitted(): void {
    this.changes = [];
  }
}
```

### 4. Repository (저장소)

```typescript
// src/repositories/AccountRepository.ts
export class AccountRepository {
  constructor(private eventStore: EventStore) {}

  save(account: Account): void {
    const changes = account.getUncommittedChanges();
    changes.forEach((event) => this.eventStore.append(event));
    account.markChangesAsCommitted();
  }

  getById(id: string): Account {
    const account = new Account(id);
    const events = this.eventStore.getEventsByAggregateId(id);
    account.loadFromHistory(events);
    return account;
  }
}
```

### 5. 사용 예제

```typescript
// src/main.ts
const eventStore = new EventStore();
const repository = new AccountRepository(eventStore);

// 새 계좌 생성
const account = new Account("ACC-001");
account.createAccount("John Doe", 1000);
account.deposit(500);
account.withdraw(200);

// 이벤트 저장
repository.save(account);

// 이벤트 스트림 확인
const allEvents = eventStore.getAllEvents();
console.log("Event Stream:", allEvents);
// [
//   AccountCreatedEvent { accountHolder: 'John Doe', initialBalance: 1000 },
//   MoneyDepositedEvent { amount: 500 },
//   MoneyWithdrawnEvent { amount: 200 }
// ]

// 과거 상태 복원
const restoredAccount = repository.getById("ACC-001");
console.log(`Balance: ${restoredAccount.balance}`); // 1300

// 특정 시점의 상태 복원 (이벤트 일부만 재생)
const accountAfterDeposit = new Account("ACC-001");
const firstTwoEvents = allEvents.slice(0, 2);
accountAfterDeposit.loadFromHistory(firstTwoEvents);
console.log(
  `Balance after deposit: ${accountAfterDeposit.balance}`
); // 1500
```

## Event Sourcing과 CQRS

[CQRS](/pattern/CQRS)와 Event Sourcing은 완벽한 조합이다.

- **Event Sourcing** - 모든 변경을 이벤트로 저장
- **CQRS** - 읽기와 쓰기 모델을 분리

함께 사용하면:

- 쓰기 모델: 이벤트 저장 (명령)
- 읽기 모델: 이벤트로부터 재구성된 뷰 (조회)
- 읽기 모델이 이벤트 스트림으로부터 항상 최신 상태 유지

## Key Points(핵심 포인트)

- **불변 로그** - 이벤트는 추가만 가능, 수정 불가 (append-only)
- **완전한 히스토리** - 모든 상태 변화의 기록이 남음
- **복잡성** - 일관성 유지, 이벤트 버전 관리 등 추가 복잡도
- **저장소 크기** - 이벤트가 계속 누적되므로 저장소 관리 필요
- **이벤트 스키마 진화** - 시간이 지남에 따라 이벤트 구조 변화 처리
