# SOLID

소프트웨어 설계원칙으로 로버트 C. 마틴에 의해 정의 되었으며 5가지 원칙의 첫글자를 따서 만듦.

1. [Single Responsibility Principle](/modules/single-responsibility-principle) (단일 책임 원칙)
2. [Open/Closed Principle](/modules/open-closed-principle) (개방-폐쇄 원칙)
3. [Liskov Substitution Principle](/modules/liskov-substitution-principle) (리스코프 치환 원칙)
4. Interface Segregation Principle (인터페이스 분리 원칙)
5. Dependency Inversion Principle (의존성 역전 원칙)
