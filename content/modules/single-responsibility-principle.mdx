# Single Responsibility Principle (단일 책임 원칙)

한 모듈은 변경되어야 하는 이유가 하나 이상 있어서는 안 된다.

자세한 설명: 모든 모듈은 단일 책임을 가져야 하며, 그 책임은 모듈에 의해 완전히 캡슐화되어야 한다. 책임은 변경되어야 하는 이유로 정의될 수 있으므로, 모듈은 변경되어야 하는 하나의 이유만 가져야 한다.

### Why

- 유지보수성을 위해 변경이 필요한 경우 하나의 모듈만 변경이 이루어져야 한다.

### How

- Curly의 법칙을 적용.

### Resources

- [단일 책임 원칙 (wikipedia.org)](https://en.wikipedia.org/wiki/Single-responsibility_principle)

### Example

```tsx
// 나쁜 예시: 하나의 컴포넌트가 여러 책임을 가짐
const UserProfile = () => {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);

  // 데이터 페칭 책임
  const fetchUser = async () => {
    setLoading(true);
    try {
      const response = await fetch("/api/user");
      const data = await response.json();
      setUser(data);
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  };

  useEffect(() => {
    fetchUser();
  }, []);

  if (loading) return <div>로딩 중...</div>;
  if (error) return <div>에러: {error}</div>;
  if (!user) return <div>사용자를 찾을 수 없습니다.</div>;

  const isValid = validateUser(user);

  return (
    <div>
      <h1>사용자 프로필</h1>
      {isValid ? (
        <>
          <h2>{user.name}</h2>
          <p>{user.email}</p>
        </>
      ) : (
        <p>유효하지 않은 사용자 데이터</p>
      )}
    </div>
  );
};

// 좋은 예시: 각 컴포넌트가 하나의 책임만 가짐
// 데이터 페칭을 담당하는 커스텀 훅
const useUserData = () => {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchUser = async () => {
      setLoading(true);
      try {
        const response = await fetch("/api/user");
        const data = await response.json();
        setUser(data);
      } catch (err) {
        setError(err.message);
      } finally {
        setLoading(false);
      }
    };

    fetchUser();
  }, []);

  return { user, loading, error };
};

// 메인 컴포넌트
const UserProfile = () => {
  const { user, loading, error } = useUserData();

  if (loading) return <div>로딩 중...</div>;
  if (error) return <div>에러: {error}</div>;
  if (!user) return <div>사용자를 찾을 수 없습니다.</div>;

  const isValid = validateUser(user);

  return (
    <div>
      <h1>사용자 프로필</h1>
      {isValid ? (
        <>
          <h2>{user.name}</h2>
          <p>{user.email}</p>
        </>
      ) : (
        <p>유효하지 않은 사용자 데이터</p>
      )}
    </div>
  );
};
```
